## S2. 한 줄 요약 — "점수를 올리기 전에, 믿을 수 있게"
- 정직한 평가 체계 먼저 + 한 번에 한 변수 → 통념(더 많이·더 크게·더 고해상도)을 데이터로 반증, **우리 누수까지 자가검증**
- 문제의 본질='커버리지' 실증 → 56cls 0.71 → 118cls 0.99 → 앙상블 **0.9994(best)**
- 점수 너머: 상용 검출→검색 설계 · 데이터 3트랙 · 멀티머신 인프라
- 화자노트: 레버마다 예측대로 올랐다는 게 이 프로젝트의 성격.

## S5. 베이스라인 — 다중 모델 공정 비교
- 동일 fold·동일 하니스: YOLO11n 0.686 · RT-DETR-l **0.737** · RetinaNet 0.58 · FasterRCNN 0.655(warmup)

## S6. 증강 — 합성의 도약, 물량은 플래토
- real만 0.726 → **+합성696 0.905 (+0.18)** → +1500/+2500 = 0.90대(플래토)
- 균형분포 강제도 개선 아님 → **데이터는 이미 충분, 다른 축 필요**
- 화자노트: SAM2 알파매트 → 구조적 Copy-Paste 합성.

## S7. 평가 신뢰도 — 3대 함정 (핵심 스토리)
- ① 단일폴드 노이즈(부트스트랩 95% CI 겹침, val 51장) → **3폴드 GroupKFold**
- ② 대회 GT 실알약 13건 누락 → val 과소평가 → 로컬 corrected 별도
- ③ '0.960 vs 0.933' = mAP@[0.50:0.95] vs [0.75:0.95] **지표범위 차이**(버그 아님)
- 화자노트: 평가를 믿을 수 있게 만든 뒤에야 이후 실험이 의미.

## S8. 용량 · 해상도 · real-copy 누수 (자기검증)
- 용량 **비단조**: 11n 0.905 → 11s **0.930(+0.025, 스윗스팟)** → 11m 0.907(base 데이터기아)
- 해상도 960 폐기(640보다 하락)
- **real-copy 누수**: 0.945 착시 → source_file 매핑 → fold1·2 누수 확정 → clean 하락, 무효화
- 화자노트: 남이 아니라 우리 결과를 스스로 의심해 잡아낸 사례.

## S9. AI Hub 조합 = 진짜 레버 · 정제·누수 관리 ('0.93 천장' 정정)
- '0.93 천장'은 좁은 슬라이스만 판 오판 → 리더보드 현실 0.98~0.99. 핵심 레버 = **AI Hub 금지-아닌 조합 real**
- **추출·정제 파이프라인** (`aihub_extract_poc.py`):

  | 단계 | 결과 |
  | --- | --- |
  | 규칙·누수 검증 | ✅ 허가 3,503조합, 금지(TL_2/TS_2) 0중복, 대회 232 0중복(누수 없음) |
  | 가치 평가 | ✅ 54/56 클래스, 희소클래스 real 대량보강(3샘플→100+) |
  | 라벨 규약 버그 | AI Hub raw `dl_idx`(249)≠`K-code`(250) 발견·수정 → `category_id=K-code` 정합 |
  | POC 추출 | 2,099장, our 4,398 인스턴스, 비-우리 3,951알 배경마스킹, 육안 정확 |

- **누수·규칙 검증(객관적)**:

  | 체크 | 결과 |
  | --- | --- |
  | aihub 700조합 ∩ 대회 232조합 | 0 (다른 조합) |
  | aihub ∩ fold0-val 조합·이미지 | 0 (누수 아님) |
  | 금지본(TL_2/TS_2) 포함 | 0 (구조적 제외+요건 충족) |

- 화자노트: real-copy는 누수로 무효화됐지만 AI Hub 조합은 교차검증 전부 0 통과 = 진짜 새 real. 부족했던 건 '진짜 real'이었음이 확정.
