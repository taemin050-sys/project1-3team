# 단일→조합 증강 설계 (Copy-Paste Synthesis) — LHK 초안

> (2) AI Hub 단일 알약을 (1) 도메인에 맞는 조합 이미지로 합성해 **희소 클래스 물량↑ + 학습셋 확장**. **train 전용**(val/test는 (1)만).

## 1. 설계를 지배하는 EDA 근거
- **(1) 도메인은 단일·통제:** 배경 **연회색 232/232**, 조명 **주백색**, **앞면·준탑다운**, 저오클루전, 이미지당 **2–4알**, **976×1280**.
- **(2)는 배경·WB 광범위** → 그대로 섞으면 테스트에 없는 조건 학습(해로움) → **(1) 도메인으로 정규화 후 합성**.
- **불균형:** 56클래스 중 48개 <20샘플, **17개 <5** → 희소 클래스 표적 보강이 핵심 명분.
- **분할 난이도 이분:** 불투명 정제(휘도 Otsu 쉬움) ↔ **반투명 연질캡슐(CV 실패 → SAM 필요)**.
- **색은 (1) 안에선 안정** → 정규화 후 color 단서 유효.

## 2. 파이프라인 (단계)
1. **필터** — (2) 단일을 `dl_idx → 우리 56클래스`로 선별(`filter_aihub`), 금지셋(TL_2/TS_2) 가드.
2. **알약 분할(알파 추출) — 하이브리드**
   - 불투명: **휘도 Otsu + 모폴로지 + 컨투어** → 마스크
   - 반투명·저대비·디퓨저: **SAM(prompt="pill")** → 마스크
   - 라우팅: 대비·마스크 견고성 **품질 게이트**로 CV 실패분만 SAM
   - **마스크 1–2px 침식**(경계 fringe 제거)
3. **WB 정규화** — 잘라낸 알약을 (1)의 **주백색 톤**으로(gray-world/white-patch, 알약 영역 기준) → `color_class` 일관성 확보.
4. **배경 생성** — (1) train 이미지의 **빈 연회색 영역을 크롭·타일링**해 정품 배경 생성(단색보다 사실적), 976×1280.
5. **배치(구성)** — 2–4알 랜덤, **비겹침 그리드 배치**(테스트 저오클루전 매칭) + 알약별 소폭 기하증강(회전·스케일, **앞면 유지**) + 약한 그림자.
6. **합성** — 알파 컴포지트 → **전역 bbox 좌표** 산출.
7. **라벨** — COCO/YOLO, `category_id = dl_idx`, SSOT 정합. **train 전용**.

## 3. 클래스 표적 전략
- **희소 17클래스(<5) 우선**, 클래스 균형 목표(예: 클래스당 최소 N표본까지).
- 조합 구성 시 (1)의 **허브약(003351) 편향은 재현하지 않음**(균형 지향).

## 4. 검증 (E5 애블레이션)
- (1) **Group K-Fold val**에서 **synth 추가 전/후 mAP@[0.75:0.95]** 비교(특히 per-class 희소).
- **과합성 리스크**: `synth:real` 비율 상한 튜닝(예: real의 2–3배 이내).
- **HITL 표본 검수** — 도메인 정합·마스크 품질 육안 확인, `decision-log`에 기록.

## 5. 재사용 자산

## 6. 확정된 결정 (2026-07-03)
1. **SAM 실행 위치** — **CV(휘도 Otsu)로 최대 커버, SAM은 어려운 케이스만**(필요 시 Colab 배치). (2) 열어 반투명 비중 확인 후 SAM 비중 조정.
2. **배경** — **(1) 연회색 빈영역 크롭 정품**(단색보다 사실적).
3. **합성 규모** — 희소 클래스 **클래스당 최소 ~50표본**까지 보강, 전체 **synth ≤ real의 2–3배**.
4. **배치 오클루전** — **대체로 비겹침(그리드)**, (1) 실제 오클루전 재확인 후 미세 조정.
5. **투입 모델** — **YOLO11(MPS)에 먼저 E5 애블레이션** → 효과 확인 후 RT-DETR(Colab).

## 7. 구현 (스크립트)
- `scripts/augment.py` — 재사용 모듈: WB정규화·CV분할·컷아웃·배경샘플·기하증강·비겹침합성·라벨. SAM은 lazy 스텁.
- `scripts/smoke_augment.py` — **(2) 없이 (1) 데이터로 엔진 검증**((1) 알약 크롭→Otsu→합성 → 시각화). (2) 도착 시 입력만 교체.

## 8. 데이터 증강 통합 전략 (real-aug + synth-aug 함께)
합성만이 아니라 **리얼 증강도 병행**한다(팀원이 real 232→2000+ 증강만으로 Kaggle 0.9 실증). 3-레버:

| 레버 | 내용 | 특징 |
| --- | --- | --- |
| **(a) 온라인 증강** | ultralytics 기본(mosaic·flip·scale·hsv·translate) — **이미 baseline에 적용** | 강도·에폭 튜닝이 레버. 무료·무한변주 |
| **(b) 리얼 Copy-Paste 합성** ⭐ | **232 real 조합의 알약을 SAM2로 잘라 블루그레이 배경에 재조합** | 완전 in-domain·real 알약·새 조합. 엔진 동일, **소스만 real**. R-aug와 S-aug의 다리 |
| **(c) AI Hub synth** | 외부 단일(56클래스)→조합 (Codex) | **희소 클래스 커버리지**(R-aug가 못 고치는 부분) |

**상보성:** (a)(b)=리얼 분포 고충실도 물량, (c)=희소 커버리지·다양성. **(b)는 Codex 파이프의 소스만 "real 232 pills"로 바꾸면 즉시 가능.**

**규율:** 한 변수씩 fold0 애블레이션 — baseline → +online↑ → +real-copypaste → +synth → 조합. R-aug 강도는 **"캐글 적합성 유지"**(기하 자유·광학 보수·색 보존). 최종 대회 모델 = **real 232 전체 + (최적 real-aug + 최적 synth)**.

> (2) AI Hub 데이터 다운로드 완료 시 착수. 그 전까지 이 설계로 스크립트 골격만 준비 가능.
