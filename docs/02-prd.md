## 1. 배경 및 문제 요약

유저가 모바일로 촬영한 이미지에서 **이미지당 최대 4개 알약의 클래스 + 바운딩 박스**를 검출하는 객체 탐지 모델을 구현한다(SRS §1). 비즈니스 맥락은 헬스잇의 복약 식별·병용 금기 등 헬스케어 부가가치이나, 본 프로젝트의 산출물은 **검출 모델과 그 재현 코드**다(SRS §11 범위 외).

**문제의 3대 특성** (설계를 지배하는 제약)
1. **저데이터** — Train 232장 (Test 842장). 과적합 위험 → 전이학습·증강·외부데이터 필수.
2. **고 IoU 지표** — **mAP@[0.75:0.95]**. 박스가 약간만 어긋나도 점수 손실 → **localization 정밀도가 1순위**.
3. **제출 제약** — 1일 10회·팀 단위(CR-01/02) → **로컬 평가로 검증 효율 극대화** 필요.

#### 5.2.3 컴포넌트별 근거 (backbone / neck / head / branch)
- **백본:** CNN(ResNet→ConvNeXt) 주력(전이·해석성), Swin/DINOv2 다양성. 사전학습은 **검출기=COCO, 백본스왑=ImageNet-21k**. 저데이터 방어 = **점진적 언프리즈 + 차등 학습률**.
- **넥:** FPN/PAN. 초소형-밀집 아님 → 극단 소형넥 불필요하나 **고해상도 레벨(P2/P3) 유지가 고-IoU 박스에 유리**.
- **헤드:** **디코플드**(cls/reg 분리) — 미세분류와 고-IoU 회귀는 요구가 다름. 박스 손실 **CIoU/DIoU + DFL**(IoU 인지 손실 = 고-IoU 직결 레버).
- **브랜치(보조):** **IoU/품질 예측 브랜치**(VarifocalNet 계열)로 고-IoU 랭킹 직접 최적화. 세그멘테이션 보조(마스크, SAM 조달)는 심화 옵션. **OCR/각인 브랜치는 SRS 범위 외** → 특징으로만 간접 활용.

#### 5.2.4 정직한 배제·보류 근거 (감점 방지: "왜 안 썼나"도 기록)
- **순수 ViT 검출:** 저데이터 부적합 → 배제(근거 있는 배제).
- **RT-DETR(정확도):** 실시간 특화라 정확도 트랙에선 DINO에 양보 → **서빙으로 재배치**(배제 아님).
- **YOLO26(주의):** 최신이라 커뮤니티 검증·안정성 축적이 덜함(재현성 채점·마감 리스크) + **E2E 직접출력은 WBF 앙상블 결합이 까다로움**(§5.4 앙상블과 상충) → **YOLO11과 통제 비교 후 트랙별 배치**.

#### 5.2.5 XAI 설계 (아키텍처별 내재 + 사후 스위트)
"모델이 **각인/형태 영역**을 보고 판단했다"를 입증: CNN/Cascade → **Grad-CAM/Eigen-CAM + D-RISE**; FCOS/CenterNet → **센터 히트맵(내재)**; DETR/DINO → **어텐션 맵(내재)**. 공통: **신뢰도 캘리브레이션·클래스별 AP·실패 케이스 시각화**(정직한 천장 서사).

> 실험 규율(docs/03 beam 1~3): 위 후보를 **풀에 올려두되 다 돌리지 않는다**. P0(YOLO26·YOLO11) 빠르게 → 선두 1~2 심화 → DINO/Cascade는 강한 사전학습 전제 하 탐색. 한 번에 한 변수. **대회=상보 계열 WBF, 서빙=단일(YOLO26).** (데이터 근거: AI Hub 단일 80장 리뷰 — 배경 변주·WB 불안정·반투명캡슐, docs/07·08 연계)

## 6. 실험 설계

### 6.1 실험 매트릭스 (초안)

| ID | 가설 | 변경점 | 주담당 | 기준 지표 |
