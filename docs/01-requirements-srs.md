## 0. 문서 개요

### 0.1 목적
본 문서는 "사진 속 최대 4개 알약의 클래스와 위치를 검출하는 모델"을 구현·평가하기 위해 **무엇을 만들어야 하는지**를 검증 가능한 요구사항 단위로 정의한다. 후속 PRD가 *어떻게 만들지*를 다루기 전, 이 문서는 *충족해야 할 조건의 집합*을 확정한다.

### 0.2 범위
요구사항은 두 층위로 구분한다.
- **제품/기술 요구사항** — 검출 모델과 데이터·인터페이스가 만족해야 할 조건
- **코스 산출물 요구사항** — 코드잇 평가·제출 규정이 요구하는 산출물과 마감

### 0.3 우선순위 표기 (MoSCoW)

| 표기 | 의미 |
| --- | --- |
| **필수(M)** | 미충족 시 프로젝트/채점 성립 불가 |
| **권장(S)** | 결과 품질·평가에 직접 기여, 강력 권장 |
| **선택(C)** | 가이드상 심화 과제, 여력에 따라 수행 |

### 0.4 출처 약어
`G`=프로젝트 가이드 · `OT`=오리엔테이션 · `K`=Kaggle(Overview/Rules) · `D`=데이터 명세

## 1. 시스템 컨텍스트 (블랙박스 정의)

```
            ┌─────────────────────────────────────┐
  입력       │        경구약제 객체 검출 모델          │   출력
 ─────────▶ │  (Object Detection: localize+classify) │ ─────────▶
 RGB 이미지   │                                      │  객체별 (category_id,
 (PNG) 1장   └─────────────────────────────────────┘   bbox[x,y,w,h], score)
                                                        이미지당 0~4개
```

- **입력:** PNG 이미지 1장(다양한 배경·조명·알약 방향 조건 포함)
- **출력:** 검출 객체 리스트 — 각 객체 = `category_id` + 바운딩 박스 `[x, y, w, h]` + 신뢰도 `score`
- **평가 경로:** 출력 → 캐글 제출 CSV → mAP@[0.75:0.95] 채점

## 3. 기능 요구사항 (FR)

| ID | 요구사항 | 우선순위 | 출처 |
| --- | --- | --- | --- |
| **FR-01** | RGB 이미지(PNG) 1장을 입력받아 이미지 내 알약 객체를 검출한다 | 필수(M) | G, K |
| **FR-02** | 이미지당 **최대 4개**의 알약을 검출한다 | 필수(M) | G, K |
| **FR-03** | 검출된 각 객체의 **클래스(category_id)**를 예측한다 | 필수(M) | G, K |
| **FR-04** | 검출된 각 객체의 **바운딩 박스 `[x, y, w, h]`**(COCO, 픽셀)를 예측한다 | 필수(M) | K, D |
| **FR-05** | 검출된 각 객체의 **신뢰도 점수(score, 0~1)**를 산출한다 | 필수(M) | K |
| **FR-06** | **Test 40개 클래스**에 대해 안정적으로 예측한다(Train에 Test 미포함 클래스가 섞여 있어도 robust) | 필수(M) | D |
| **FR-07** | 후처리(NMS, score threshold)로 중복 박스를 제거하고 출력 객체 수를 통제한다 | 필수(M) | (지표 도출) |
| **FR-08** | 검출 결과를 캐글 제출 **CSV 포맷으로 변환·출력**한다 | 필수(M) | K |
| **FR-09** | 검출 결과를 이미지 위에 **시각화(박스+클래스+score)**하여 정성 검증한다 | 권장(S) | G |
| **FR-10** | 모델 성능을 정량 지표(mAP, 클래스별 AP 등)로 산출·비교한다 | 필수(M) | G |
| **FR-11** | (심화) 후속 피드백 데이터로 **재학습 가능한 파이프라인**을 제공한다 | 선택(C) | G, 목표 |
| **FR-12** | (심화) 추론 **API/서빙 인터페이스**를 제공한다 *(최중열 서빙 워크스트림)* | 선택(C) | 목표 |

## 13. 용어 정의 (Glossary)

| 용어 | 정의 |
| --- | --- |
| Object Detection | 이미지 내 객체의 **위치(bbox) + 클래스**를 동시에 예측하는 과제 |
| bbox | Bounding Box, COCO 표기 `[x, y, width, height]`(좌상단 기준 픽셀) |
| mAP@[0.75:0.95] | IoU 0.75~0.95 구간 평균 AP. 임계값이 높아 **박스 정밀도**가 점수에 직결 |
| IoU | 예측 박스와 정답 박스의 교집합/합집합 비율 |
| NMS | 중복 검출 박스를 억제하는 후처리 |
| COCO format | 객체 검출 표준 주석 포맷(JSON) |
| 존재/유효(bbox) | 존재=`bbox` 키 있고 비어있지 않음 / 유효=리스트 길이 정확히 4 |
