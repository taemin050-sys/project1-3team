## 2. 문제 정의 〔팀 공식 · 홍우석 작성〕

현대인은 과학의 발전으로 수명이 늘었지만, 그로 인해 여러 가지 병에 노출되고 있습니다. 특히 고령자나 만성 질환자의 경우 복용 중인 약이 여러 가지인 경우가 많아, 외형과 이름이 비슷한 약들의 종류를 정확히 기억하거나 구분하는 데 어려움을 겪는 경우가 많습니다.

본 프로젝트는 이러한 문제를 해결하기 위해, 유저가 모바일 기기로 복용 중인 약을 촬영한 이미지에서 알약의 위치와 종류를 자동으로 인식하는 **AI 객체 탐지 모델**을 구현하는 것을 목표로 합니다. 모델은 하나의 이미지 안에 포함된 **최대 4개 알약의 위치를 각각 바운딩 박스로 검출**하고, 검출한 알약의 **클래스를 예측**합니다.

이를 통해 유저는 촬영한 이미지를 기반으로 자신이 복용 중인 약을 보다 쉽게 식별할 수 있게 되며, 나아가 **복용 정보 제공·병용 금기 확인·복약 관리 서비스** 등 헬스케어와 연계된 서비스를 제공받을 수 있게 됩니다.

이는 단순 이미지 분류가 아니라, **여러 개의 알약을 동시에 탐지하고 각각의 위치와 클래스를 예측해야 하는 객체 탐지 문제**로 정의할 수 있습니다. 모델은 알약의 색상·모양·크기·각인·각도·조명 변화 등 다양한 시각적 조건에서도 안정적으로 알약을 구분할 수 있어야 합니다.

> **〔개인 보강 — 기술 정의〕**
> 위 정의를 모델 관점으로 좁히면: 입력 = RGB 이미지 1장, 출력 = 객체별 `(category_id, bbox[x,y,w,h], score)` 리스트(이미지당 0~4개). **Test 40개 클래스**가 예측 대상이며, Train에는 Test에 없는 클래스가 일부 섞여 있으므로 "보지 못한 클래스에도 흔들리지 않는 robustness"가 암묵 요구사항.

## 3. 목표 〔팀 공식 · 장한빈 작성〕

본 프로젝트는 사용자가 촬영한 알약 이미지를 분석하여 정확한 약품 정보를 제공함으로써, 사용자의 안전한 복약 환경을 조성하고 헬스잇(Health-Eat) 서비스의 핵심 가치를 실현하는 데에 목적이 있다. 더불어 단순한 기술 구현을 넘어, 팀 단위의 협업과 엔지니어링 역량 강화를 통하여 성장을 도모한다.

### 정량적 목표
- **다중 객체 검출** — 단일 이미지 내 최대 4개의 알약을 동시에 인식하고, 각각의 Bounding Box를 정확하게 표기한다.
- **정확도 확보** — 데이터셋 기반 검증을 통해 알약 식별 모델의 **mAP `nn`% 이상**을 달성한다.
- **실시간성 및 편의성** — 촬영 즉시 결과를 확인할 수 있도록 모델의 추론 속도를 최적화한다.

### 구현 목표
- **데이터셋 구축** — 제공된 train set과 더불어 다양한 조명 환경과 배경이 포함된 알약 데이터셋을 확보하고, 전처리를 거쳐 학습용 데이터를 구축한다.
- **모델 고도화** — 여러 Object Detection 알고리즘을 활용하여 알약의 고유한 외형(color, thick, shape, line 등) 특징을 추출하는 모델을 설계한다.
- **CI/CD 환경 수립** — 추후 수집되는 피드백 데이터를 활용해 모델을 재학습하고, 성능을 지속적으로 업데이트할 수 있는 파이프라인을 구축한다.

### 성장 목표
- **팀 단위 협업 역량 강화** — 실제 엔지니어링 환경에서의 협업 프로세스를 경험하고 역량을 키운다.
- **완결형 성과 창출** — 하나의 결과물을 끝까지 완성하며 문제 해결 능력과 책임감을 함양한다.

> **〔개인 보강 — `nn`% 목표 설정 가이드〕**
> 캐글 평가지표는 **mAP@[0.75:0.95]** (표준 COCO 0.5:0.95보다 IoU 임계값이 높음). 따라서 *절대 수치가 mAP@0.5 기준보다 낮게 나오는 게 정상*이므로 `nn`% 목표는 **베이스라인 1차 제출 점수를 본 뒤** 현실적으로 확정 권장. 목표는 "특정 절대치"보다 **베이스라인 → 중간발표 → 최종으로 이어지는 우상향 곡선과 그 원인 설명**에 둠(가이드상 순위는 평가 미반영).
> "실시간성·추론 속도 최적화"와 "CI/CD"는 최중열 님 **서빙/배포 워크스트림**과 직접 연결됨.

## 4. 데이터 계획 〔개인 보강 — 홍우석(DE) 협의용〕

> 공식 데이터 계획은 홍우석 님이 Notion에서 작성. 아래는 Model Architect 입력과의 인터페이스 정리.

### 4.1 데이터 현황 (캐글 명세)

| 구분 | 내용 |
| --- | --- |
| 형식 | 이미지 **PNG**, 주석 **COCO 포맷(JSON)** |
| 규모 | 총 1,837 파일 / 1.93 GB |
| Train | 이미지 232장, annotation 114 디렉토리 |
| Test | 이미지 842장 |
| 클래스 | **Test 40개.** Train에는 Test에 없는 클래스 일부 포함 |
| bbox 형식 | COCO `[x, y, width, height]` |
| 메타데이터 | 모양·색상·각인·제형·배경색·조명·방향 등 풍부 |

> **저데이터 시그널:** train 232장 < test 842장 → 전이학습·증강·외부 데이터가 사실상 필수.

### 4.2 EDA / 전처리 / 분할
- COCO 파싱 → bbox 유효성 검사(길이 4 리스트), 클래스 분포·이미지당 객체 수 분포
- **Test 40클래스 기준 category_id 매핑표 확정** (제출 0점 방지의 핵심)
- bbox 크기·종횡비·위치 분포 → 입력 해상도/앵커 근거
- 리사이즈 + 정규화(pretrained 통계), **train/val 분할**(232장 한정 → K-Fold 검토, 리더보드 과의존 방지용 내부 검증선)

### 4.3 증강 (mAP@high-IoU 친화)
- 기하: 회전·플립(방향 무작위), 스케일/이동 — **bbox 동기 변환 필수**(Albumentations)
- 광학: 밝기·대비(조명). **색상은 약 구분 단서 → color jitter 보수적으로**
- Mosaic/MixUp은 효과 검증 후 선택. **박스 정확도 해치는 증강은 배제**

### 4.4 외부 데이터 (최대 점수 레버) + ⛔ 금지
캐글 규칙상 외부 데이터 사용 가능, AI Hub 추가 데이터 권장(클래스 정보 상이 → 확인 필수). 통합 시 **Test 40클래스 매핑 정합성 검증** 후 합류.

> ⛔ **절대 사용 금지:** `경구약제조합 5000종 > TL_2_조합.zip`, `TS_2_조합.zip`
> (대회 train/test 세팅에 사용 → 점수 비정상 산출, **추적·적발 대상**)

> ⚠️ **〔리스크 연결〕** 팀원 각자 외부 데이터를 수집·병합하면 **편향** 발생 가능(이태민 님 리스크 항목과 동일). → 수집 출처·클래스·전처리 규격을 홍우석 님이 **단일 기준으로 통제**하고, 병합 전 분포 점검 필수.

## 5. 모델 계획 〔개인 — Model Architect 주관 (이형기)〕

> **본 섹션이 본인의 핵심 책임 영역.** 두 Experimentation Lead(장한빈·최중열)에게 넘길 학습/실험 인터페이스를 명확히 정의하는 것이 목표.

### 5.1 지표 해석 → 선정 1순위 기준
**mAP@[0.75:0.95] = 박스 정밀도 게임.** localization이 강한 모델 + 정밀 박스 회귀를 우선.

### 5.2 후보 모델

| 모델 | 유형 | 선택 근거 |
| --- | --- | --- |
| **YOLOv8 / YOLO11 (Ultralytics)** | 1-stage | 베이스라인 1순위 — 전이학습·문서·속도 최적, 빠른 반복 |
| **Faster R-CNN (torchvision)** | 2-stage | 정밀 localization → 고 IoU 지표 유리, 정확도 비교군 |
| **RetinaNet** | 1-stage | Focal Loss로 클래스 불균형 대응 |
| **RT-DETR** | Transformer | 앵커프리·후처리 단순, 여력 시 도전 |

> 가이드 언급 VggNet/ResNet은 **백본**으로 검출 헤드와 결합.

### 5.3 선택 기준
1. mAP@[0.75:0.95] (최우선) 2. 저데이터 전이학습 안정성 3. 학습/추론 속도(2.5주 반복) 4. 팀 환경 호환(MPS·Colab 등 통일)

### 5.4 개선 전략 (베이스라인 이후)
- COCO pretrained 전이학습(필수) · 입력 해상도 상향
- **후처리 튜닝:** NMS IoU·**score threshold** 스윕(mAP는 recall 적분 → 임계값이 점수 직결)
- TTA · 모델/폴드 **앙상블** · 외부 데이터 전후 ablation으로 효과 정량화

### 5.5 Model Architect → 팀 인터페이스 (산출물)
- 표준화된 **학습 스크립트 + config 템플릿**(seed 고정·재현성) → Experimentation Lead가 실험 변주
- **공통 평가 루틴**(내부 val mAP, 클래스별 AP, 오검출 시각화) → 실험 비교 기준 통일
- **제출 변환기**(모델 출력 → 캐글 CSV 포맷) → Data/Experimentation 공용
