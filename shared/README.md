# shared/ — 경기 규칙 (전원 공용, 얇게)

프로젝트 시작부터 **전원이 공유·재사용**하는 최소 규칙. 마일스톤 승격과 무관하게 처음부터 공용이다.
개인 방식은 자유지만, **여기(채점기·가드·계약)는 모두 같은 것을 써야 내부 경쟁이 공정**해진다.

| 모듈 | 역할 | 왜 공용인가 |
| --- | --- | --- |
| `guards/banned.py` | 금지 데이터 가드(조합/TL_2/TS_2) | **양보 불가** — 한 명이 쓰면 팀 0점 |
| `eval/local_map.py` | mAP@[0.75:0.95] 로컬 심판 | 같은 자로 재야 비교가 공정 |
| `submit/make_submission.py` | 제출 CSV·포맷 검증(IR-06) | 잘못된 제출=0점 방지 |
| `ssot/build_drug_master.py` | class_map·drug_master(category 계약) | 같은 클래스 체계여야 비교 가능 |
| `ssot/build_crosswalk.py` | 품목기준코드↔category_id | 외부/오토라벨 데이터 합류 계약 |
| `env.py` | 디바이스 자동·시드 | 환경 무관 재현 |

- 실행: 레포 루트에서 `python -m shared.<...>` (코드 위치 맵: 루트 README).
- 규약: **이 폴더는 함부로 갈아엎지 않는다.** 변경은 전원 합의(공정성 코어).
